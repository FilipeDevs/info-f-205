# Exercice 3 — Nombre de bits pour garantir précision et plage de représentation

## 🧱 Brique 1 — Distance relative entre deux nombres consécutifs

Dans $\mathcal{F}(b, t, L, U)$, deux nombres consécutifs ne sont pas espacés
de la même manière partout. Cet espacement **dépend de l'exposant** $e$ du nombre.

On a vu que la distance absolue entre deux nombres consécutifs vaut $b^{e-t}$.
La **distance relative**, elle, est définie comme :

$$\eta(x_i) = \frac{|x_{i+1} - x_i|}{|x_i|} = \frac{1}{m[x_i]}$$

où $m[x_i]$ est la mantisse de $x_i$. Puisque la mantisse vérifie
$b^{t-1} \leq m \leq b^t - 1$, on peut montrer que :

$$\frac{1}{b} \cdot \varepsilon \leq \eta(x_i) \leq \varepsilon \quad \text{où} \quad \varepsilon = b^{1-t}$$

La distance relative maximale entre deux nombres consécutifs est donc :

$$\boxed{\varepsilon = b^{1-t}}$$

C'est ce qu'on appelle l'**epsilon machine**.

> 💡 **Intuition :** Peu importe si le nombre est grand ou petit, l'écart
> *relatif* entre deux nombres consécutifs reste toujours du même ordre.
> C'est la grande force de la virgule flottante : la précision relative
> est uniforme sur toute la plage de représentation.

## 🧱 Brique 2 — Overflow et Underflow

La machine ne peut représenter qu'une **plage finie** de nombres, bornée par :

$$\text{realmin} = b^{L-1} \qquad \text{(le plus petit nombre non-nul representable)}$$
$$\text{realmax} = b^U(1 - b^{-t}) \qquad \text{(le plus grand nombre representable)}$$

- **Underflow** : le résultat d'un calcul est **trop petit** (en valeur absolue)
  pour être représenté → la machine le remplace par 0, ce qui est faux.

- **Overflow** : le résultat d'un calcul est **trop grand** pour être représenté
  → la machine renvoie $\pm\infty$, ce qui est faux.

> 💡 **Exemple :** Sur une machine avec $b=10$, $L=-5$, $U=5$ :
> - $\text{realmin} = 10^{-6}$, donc $10^{-7}$ provoquerait un underflow
> - $\text{realmax} \approx 10^5$, donc $10^6$ provoquerait un overflow

Pour éviter l'underflow sur un nombre $x_{\min}$ et l'overflow sur $x_{\max}$ :

$$b^{L-1} \leq x_{\min} \implies L \leq 1 + \log_b(x_{\min})$$
$$b^U \geq x_{\max} \implies U \geq \log_b(x_{\max})$$

## 🧱 Brique 3 — Comment compter les bits nécessaires

Un nombre à virgule flottante en base $b=2$ est stocké en 3 parties :

    ┌───────┬───────────────────┬──────────────────────┐
    │ Signe │     Exposant      │       Mantisse       │
    │ 1 bit │    ? bits         │       ? bits         │
    └───────┴───────────────────┴──────────────────────┘

**Bits pour le signe :** toujours **1 bit** (0 = positif, 1 = négatif).

**Bits pour la mantisse :**
En représentation normalisée, le premier chiffre de la mantisse est
**toujours 1** en base 2 (car $x_1 \neq 0$ et les seuls chiffres sont 0 et 1).
On ne le stocke donc pas — c'est le "hidden bit". Il faut donc $t - 1$ bits.

**Bits pour l'exposant :**
L'exposant peut prendre toutes les valeurs entières de $L$ à $U$,
soit $U - L + 1$ valeurs distinctes. Pour représenter $N$ valeurs distinctes,
il faut au moins :

$$\lceil \log_2(N) \rceil = \lceil \log_2(U - L + 1) \rceil \text{ bits}$$

**Total :**

$$\boxed{\text{bits totaux} = 1 + (t-1) + \lceil \log_2(U - L + 1) \rceil}$$

## 🔧 Recette à l'examen

1. **Contrainte sur $t$** (précision) : écrire $\varepsilon = b^{1-t} \leq$ seuil,
   résoudre pour $t$, arrondir vers le haut.

2. **Contrainte sur $U$** (pas d'overflow sur $x_{\max}$) :
   $U \geq \log_b(x_{\max})$, arrondir vers le haut.

3. **Contrainte sur $L$** (pas d'underflow sur $x_{\min}$) :
   $L \leq 1 + \log_b(x_{\min})$, arrondir vers le bas.

4. **Bits mantisse** : $t - 1$ (le premier bit est toujours 1, on ne le stocke pas).

5. **Bits exposant** : $\lceil \log_2(U - L + 1) \rceil$.

6. **Total** : $1 + (t-1) + \lceil \log_2(U - L + 1) \rceil$.

## ✏️ Application — Exercice 3

**Énoncé :** Soit $\mathcal{F}(b, t, L, U)$ une représentation à virgule flottante
en base $b = 2$. On impose que la distance relative entre deux nombres consécutifs
de $\mathcal{F}(b, t, L, U)$ soit majorée par $10^{-8}$, tandis que les nombres
$\pm 10^{\pm 64}$ soient représentés sans overflow ou underflow.
Combien de bits (symboles binaires) sont nécessaires pour atteindre ces objectifs ?

**Étape 1 : Contrainte sur $t$ (précision)**

$$\varepsilon = b^{1-t} = 2^{1-t} \leq 10^{-8}$$

$$1 - t \leq \log_2(10^{-8}) = -8 \log_2(10) \approx -26.58$$

$$t \geq 27.58 \implies t = 28$$

Bits pour la mantisse (hidden bit) : $t - 1 = \mathbf{27 \text{ bits}}$

**Étape 2 : Contrainte sur $U$ (pas d'overflow sur $10^{64}$)**

$$2^U \geq 10^{64} \implies U \geq 64 \cdot \log_2(10) \approx 212.6 \implies U = 213$$

**Étape 3 : Contrainte sur $L$ (pas d'underflow sur $10^{-64}$)**

$$2^{L-1} \leq 10^{-64} \implies L - 1 \leq -64 \cdot \log_2(10) \approx -212.6 \implies L = -212$$

L'exposant couvre donc l'intervalle $[-212,\ 213]$, soit $213 + 212 + 1 = 426$ valeurs... 

> ⚠️ **Note :** Le corrigé prend $L = -213$ pour avoir une plage symétrique
> $[-213,\ 213]$, ce qui donne $213 + 213 + 1 = 427$ valeurs.
> En pratique, on prend la borne la plus conservative.

Bits pour l'exposant :

$$\lceil \log_2(427) \rceil = \lceil 8.74 \rceil = \mathbf{9 \text{ bits}}$$

**Étape 4 : Total**

$$\text{bits totaux} = \underbrace{1}_{\text{signe}} + \underbrace{27}_{\text{mantisse}} + \underbrace{9}_{\text{exposant}} = \boxed{37 \text{ bits}}$$

### Réponse à écrire à l'exam :
> La contrainte de précision $2^{1-t} \leq 10^{-8}$ impose $t = 28$, soit $27$ bits
> pour la mantisse (hidden bit). Les contraintes d'overflow/underflow imposent
> $U = 213$ et $L = -213$, soit $427$ valeurs d'exposant à représenter sur
> $\lceil \log_2(427) \rceil = 9$ bits. Le total est $1 + 27 + 9 = \mathbf{37}$ bits.
